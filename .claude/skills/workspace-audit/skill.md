# Workspace UI Audit

워크스페이스 포탈의 현재 UI를 분석하고 구체적인 개선 명세를 작성하는 스킬.

## 분석 프레임워크

### 1. 구조 분석
- 컴포넌트 트리와 렌더 흐름 파악
- 섹션별 코드 라인 수와 복잡도 측정
- Server Component / Client Component 경계 확인

### 2. 레이아웃 평가 (6개 축)

| 축 | 점검 항목 |
|----|----------|
| **시각 위계** | 제목/부제/본문 크기 대비, 색상 대비, 여백 리듬 |
| **정보 밀도** | 한 화면에 보이는 정보량, 스크롤 깊이, 그룹핑 |
| **반응형** | sm/md/lg/xl 브레이크포인트별 레이아웃 변화 |
| **다크모드** | CSS 변수 사용, dark: 접두사, oklch 색상 |
| **인터랙션** | 클릭/호버/포커스 피드백, 로딩 상태, 전환 애니메이션 |
| **접근성** | aria 속성, 키보드 탐색, 색상 대비 비율 |

### 3. shadcn/ui 활용도
- 사용 가능하지만 미사용 중인 shadcn/ui 컴포넌트 식별
- 커스텀 구현을 shadcn/ui로 대체할 수 있는 부분

### 4. Tailwind v4 최적화
- 인라인 스타일 → Tailwind 유틸리티 전환 가능 부분
- 반복되는 클래스 패턴 → `@apply` 또는 컴포넌트 추출

## 출력 형식

`_workspace/01_ui_audit_spec.md`에 다음 구조로 작성:

```markdown
# 워크스페이스 UI 개선 명세

## 현재 상태 요약
- 컴포넌트 구조 다이어그램
- 주요 수치 (총 라인 수, 섹션 수, 상태 변수 수)

## 문제점 분석
- [P1] 심각한 UX 이슈
- [P2] 레이아웃/구조 이슈
- [P3] 개선 권장 사항

## 개선 명세
### 개선 1: {제목}
- 현재: {현재 상태}
- 개선: {구체적 변경 사항}
- 구현: {파일명, 위치, 코드 수준 가이드}
- 우선순위: P1/P2/P3

## 구현 순서
1. ...
2. ...
```

## 주의사항

- `work-history-portal.tsx`가 2000줄 이상이므로 섹션별로 나눠 읽는다
- 설치된 shadcn/ui 컴포넌트 목록은 `components/ui/` 디렉토리에서 확인
- Tailwind v4 테마 변수는 `app/globals.css`에서 확인
- 기존 기능을 보존하는 범위 내에서 개선을 설계한다
