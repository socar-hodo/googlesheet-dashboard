# Milestone v1.1: 분석 도구 강화

**Status:** ✅ SHIPPED 2026-03-01
**Phases:** 6-8
**Total Plans:** 7

## Overview

기간 필터링, 데이터 내보내기, KPI 스파크라인으로 대시보드 분석 활용도를 높였다. v1.0 MVP 위에 사용자가 원하는 기간의 데이터를 즉시 조회하고 파일로 내보내며 추이를 한눈에 파악할 수 있는 3가지 기능을 추가했다.

## Milestone Summary

**Key Decisions:**

- xlsx CDN tarball 설치: npm 레지스트리 0.18.5 보안 취약점(Denial of Service, Prototype Pollution) 회피 — cdn.sheetjs.com 0.20.3 사용
- DashboardContent가 period 상태 소유: 기간 필터 범위가 KPI/차트/테이블 전체 영향 — 단일 소유자 패턴
- toISODate 직접 구현: Date.toISOString() UTC 기준 날짜 오류 방지 (로컬 시간대 보장)
- var(--chart-1) CSS 변수 직접 사용: useTheme import 없이 다크/라이트 테마 자동 전환
- KpiCards fullData prop: sparkline에 필터링 전 전체 이력 사용, 기간 필터에 관계없이 최근 7일 트렌드 표시
- Google Sheets 날짜 정규화를 parseDailySheet에서 처리: 모든 컨슈머가 ISO 형식을 받도록 보장
- vitest 선택: ESM/TypeScript 네이티브, Next.js 독립 실행, 별도 babel 설정 불필요

**Issues Resolved:**

- Google Sheets 날짜 형식 불일치 (`"2026. 2. 26"` vs `"2026-02-26"`) — filterDailyByPeriod 0건 반환 버그 수정
- sparkline 기간 첫날 데이터 부족(1개 데이터점) — fullData prop으로 전체 이력 사용
- mock-data.ts에 이번 달 데이터 없어 기본 기간 렌더링 실패 — 2026-03 데이터 추가
- Turbopack 캐시 미반영 — `.next/dev/cache/turbopack` 삭제 후 재시작으로 해결

**Issues Deferred:**

- 사용자 지정 날짜 범위 필터 (달력 피커) — v2 범위
- 차트 이미지(PNG) 내보내기 — v2 범위

**Technical Debt Incurred:**

- `npm run lint` 전체 실행 시 `.claude/get-shit-done/bin/*.cjs`와 `components/dashboard/update-timestamp.tsx`의 기존 에러 잔존 (이번 플랜 범위 외)
